jJSP/Servlet笔记

动态网页的基本知识

Tomcat

​ Apache基金会的Jakarta项目中的一个核心项目（http://tomcat.appache.org），这是是一个Web的容器，是JSP的容器。

目录 说明
/bin 存放各种平台下用于启动和停止Tomcat的脚本文件
/conf 存放Tomcat服务器的各种配置文件
/lib 存放tomcat服务器所需的各种jar文件
/logs 存放tomcat的日志文件
/temp tomcat运行时用于存放临时文件
/webapps web应用的发布目录
/work tomcat把由jsp生成的servlet存放于此目录下

启动与关闭Tomcat

bin文件加下

startup.bat——启动tomcat

Startdown.bat——关闭tomcat

验证和访问

在浏览器中输入”http://localhost:8080/“进行访问，成功进入的话就代表启动成功

Web目录结构

此图片上的WebContent相当与正规写法的Web的目录

目录 作用
WebContent 根目录下的文件对客户端都可以访问，包括jsp、HTML、js、CSS、图片等。
META-INF 存在程序入口相关信息, 每个jar 都会有这个文件夹,里面的 MANIFEST文件 记录这些信息
WEB-INF 文件夹是一个非常安全的文件，在页面中不能直接访问其中的文件，必须通过web.xml文件对要访问的文件进行相应映射才能访问。
WEB-INF/lib 存放Web程序使用的JAR文件

一个标准的jsp界面组成

jsp的指令元素

通过设置指令元素中的属性，在jsp运行时，控制jsp页面的某些特性

jsp指令“<<%@”开始，以“%>>”结束

jsp指令集中的page指令

针对当前页面进行设置的一种指令，位于jsp页面的顶端

page指令的语法：

<%@ page 属性1=”属性值” 属性2=”属性值1，属性值2” …..属性n=”属性值”%>

上图中的指令是page的默认属性设置

page指令常用属性
属性 描述
language 指定jsp页面使用的脚本语言，默认为“java”
Import 通过该属性引用脚本语言中使用到的类文件
contentType 用于定义MEMI类型和jsp页面回应时采用的字符编码格式，默认为“text/html;charset=ISO-8859-1”

language现在的jsp脚本只能使用java脚本语言

Import在实际开发中使用比较的频繁，Import属性可以在jsp文件的脚本片断中引用类也就是一个导包的意思，也可以导入多个包用逗号隔开

contentType该属性在开发中非常的重要，常用与解决中文乱码问题，这个属性就是设置页面的编码格式和类型。“text/html;”表示该页面以HTML格式显示。“charset=ISO-8859-1”表示字符的编码格式是”ISO-8859-1“

jsp的执行过程

JSP 页面在第一次被访问时，JSP引擎将它翻译成一个Servlet源程序（可识别的Java源代码），接着再把这个Servlet源程序（可识别的Java源代码）编译成Servlet的class类文件（编译成可执行的字节码文件），然后再由WEB容器（Servlet引擎）像调用普通Servlet程序一样的方式来装载和解释执行这个由JSP页面翻译成的Servlet程序就节省了前两步的繁琐，这样的话第一次编译时加载比较慢

Jsp数据交互

JSP内置对象

JSP内置对象是指不需要做任何声明就可以创建的对象

JSP out内置对象

out用于浏览器的输出数据，最常用的方法是print();，该方法用于页面显示字符串信息。

JSP request内置对象

request用于获取客户端发送的请求

方法名 说明
String getParameter(String name) 根据页面表单组件名称获取页面提交数据
String[ ] getParameterValues(String name) 获取一组以相同名称命名的表单组件提交的数据
void setCharacterEncoding(String charset) 指定每一个请求的编码，在调试request.getParameter() 方法之前进行设定，可以用与解决中文乱码问题
RequestDispatcher getRequestDdispatcher(String path) 返回一个javax.servlet.RequestDispatcher对象，该对象的forward()方法用与转发请求
